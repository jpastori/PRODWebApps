Procedure 1: Oracle System Server Software Startup
(S99zoracle start).


      The Oracle system server software startup procedure is done automatically when the GANDALF.SDSU.EDU server is started. It can also be done manually from the command line in a secure shell (ssh) session:
    1. Start the secure shell (ssh) software to create a telnet style session.

    2. Login as user oracle. (See Barry, Carol, John or Ron for current user account passwords.)

    3. Change to user root with the su command: su [return], Enter root password [return].

    4. Change to the /etc/rc3.d directory: cd /etc/rc3.d [return].

    5. Run the Oracle System Server software startup script S99zoracle: ./S99zoracle start [return].

    6. Check the status of the startup procedure by using the psgrep command: psgrep oracle [return].

    7. Log out of the secure shell (ssh) software by typing: exit [return] and closing the window.



Procedure 2: Oracle System Server Software Shutdown
(S99zoracle stop).


      The Oracle system server software shutdown procedure is done automatically when the GANDALF.SDSU.EDU server is shutdown. It can also be done manually from the command line in a secure shell (ssh) session:
    1. Start the secure shell (ssh) software to create a telnet style session.

    2. Login as user oracle. (See Barry, Carol, John or Ron for current user account passwords.)

    3. Change to user root with the su command: su [return], Enter root password [return].

    4. Change to the /etc/rc3.d directory: cd /etc/rc3.d [return].

    5. Run the Oracle System Server software shutdown script S99zoracle: ./S99zoracle stop [return].

    6. Check the status of the startup procedure by using the psgrep command: psgrep oracle [return].

    7. Check that all Oracle shared memory segments have been released by typing: ipcs [return].

    8. Log out of the secure shell (ssh) software by typing: exit [return] and closing the window.

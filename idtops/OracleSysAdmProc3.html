Procedure 3: Individual Oracle Instance Manual Start up/Shut down (sqlplus).


      SQLPLUS is a program provided by the Oracle System Server software that allows us to run Structured Query Language (SQL) commands directly against a database instance from the command line in a secure shell (ssh) session:
    1. Start the secure shell (ssh) software to create a telnet style session.

    2. Login as user oracle. (See Barry, Carol, John or Ron for current user account passwords.)

    3. If you are not currently using the TCSH shell, start it by typing: tcsh [return].

    4. Set the instance you want to run SQL commands against by typing: setenv ORACLE_SID PRODAPPS [return]. (Production instance name.)
      If you want to run against the development instance replace PRODAPPS with devapps. PRODAPPS must be uppercase and devapps must be lower case for the name to be set properly.

    5. Start the SQLPLUS program without the login option by typing: sqlplus /nolog [return]. You will see the line prompt change to sql>.

    6. Next, you must connect to the oracle instance by typing: connect / as sysdba [return]. This command gives you superuser privileges in Oracle.

    7. To startup the database instance you set in step 4, type: startup open [return]. Information will display on your screen showing the progress of the startup process.

    8. You can type any legal SQL commands to query or alter data in any user table in the current instance when the database instance is open.

    9. To shutdown the database instance you set in step 4, type: shutdown immediate [return]. Information will display on your screen showing the progress of the shutdown process.

    10. To exit SQLPLUS type: quit [return].

    11. Log out of the secure shell (ssh) software by typing: exit [return] and closing the window.

Procedure 7: Restore Gandalf Firewall port parameter file from backup copy.

      This procedure fixes the firewall parameter file override problem created when the redhat-config-securitylevel GUI program is used to update Gandalf’s firewall settings. The current version of the gui program does not allow for entry of other ports such as those needed for the Oracle Listener program. The Oracle Listener listens on Port 1521 for requests to the Oracle Development Instance, devapps. The Oracle Listener listens on Port 1522 for requests to the Oracle Production Instance, PRODAPPS.

      To copy the backup firewall parameter file to the copy used by the system:
    1. Start the secure shell (ssh) software to create a telnet style session.

    2. Login as user oracle. (See Barry, Carol, John or Ron for current user account passwords.)

    3. Change to user root with the su command: su [return], Enter root password [return].

    4. Change to the sysconfig directory by typing: cd /etc/sysconfig [return].

    5. Copy the backup firewall parameter file to the copy used by the system by typing: cp redhat-config-securitylevel.bkup redhat-config-securitylevel [return].

    6. Log out of the secure shell (ssh) software by typing: exit [return] and closing the window.



Procedure 8: Check available disk space on the Gandalf server.


      One of the major issues to keep track of is available disk space on the /oracle/ Oracle System server software volume, the /ora/data/ oracle table data volume, and the /ora/archive/ data volume.

      To check available disk space:
    1. Start the secure shell (ssh) software to create a telnet style session.

    2. Login as user oracle. (See Barry, Carol, John or Ron for current user account passwords.)

    3. Display the available disk space on each system volume by typing: df –k [return].
